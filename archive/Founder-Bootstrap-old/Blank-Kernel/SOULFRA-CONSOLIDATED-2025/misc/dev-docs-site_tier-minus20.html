Quick Start Guide

Get up and running with Soulfra in 5 minutes. This guide will show you how to protect your first piece of data.

⚡ 3-Line Data Protection

import { Soulfra } from '@soulfra/sdk';
const soulfra = new Soulfra();
const protected = await soulfra.protect(yourData);

That's it! Your data is now mathematically unbreachable.

Installation

Package Managers

npm

npm install @soulfra/sdk

yarn

yarn add @soulfra/sdk

pnpm

pnpm add @soulfra/sdk

CDN

For quick prototyping, you can use our CDN:

<script src="https://cdn.soulfra.com/sdk/latest.min.js"></script>

Requirements

  • Node.js 14+ (for Node.js environments)
  • Modern browsers (Chrome 90+, Firefox 88+, Safari 14+)
  • React 16.8+ (for React integration)

Authentication

API Keys

Get your API key from the Soulfra Dashboard.

Environment Variables

# .env file
SOULFRA_API_KEY=your_api_key_here

Direct Configuration

const soulfra = new Soulfra({
  apiKey: 'your_api_key_here'
});
Security Note: Never commit API keys to version control. Always use environment variables in production.

protect()

Protects any data by fragmenting it into mathematically unbreachable pieces.

Syntax

soulfra.protect(data, options?)

Parameters

Parameter Type Description
data any The data to protect (objects, strings, buffers, etc.)
options object Optional configuration

Options

{
  securityLevel: 'maximum' | 'standard' | 'basic',
  retentionPeriod: 'forever' | '7years' | '1year',
  shareWith: ['email@example.com'],
  compliance: 'HIPAA' | 'GDPR' | 'SOC2'
}

Returns

Returns a ProtectedData object:

{
  id: 'sf_1234567890',
  protected: true,
  size: 1024,
  fragments: 5,
  retrieveUrl: 'https://api.soulfra.com/retrieve/sf_1234567890',
  retrieve: Function
}

Example

// Protect sensitive user data
const userData = {
  ssn: '123-45-6789',
  creditCard: '4111-1111-1111-1111',
  medical: { condition: 'Diabetes', medication: 'Insulin' }
};

const protected = await soulfra.protect(userData, {
  securityLevel: 'maximum',
  compliance: 'HIPAA'
});

console.log('Protected ID:', protected.id);
// Save protected.id to your database instead of the actual data

React Integration

Complete example of using Soulfra in a React application.

Installation

npm install @soulfra/sdk @soulfra/react

Provider Setup

// App.jsx
import { SoulfraPovider } from '@soulfra/react';

function App() {
  return (
    <SoulfraProvider apiKey={process.env.REACT_APP_SOULFRA_KEY}>

    </SoulfraProvider>
  );
}

Hook Usage

// ProtectForm.jsx
import { useSoulfra } from '@soulfra/react';

function ProtectForm() {
  const { protect, isProtecting } = useSoulfra();
  const [data, setData] = useState('');
  const [protectedId, setProtectedId] = useState(null);
  
  const handleSubmit = async (e) => {
    e.preventDefault();
    const result = await protect({ content: data });
    setProtectedId(result.id);
    setData(''); // Clear sensitive data
  };
  
  return (
    <form onSubmit={handleSubmit}>
      <textarea
        value={data}
        onChange={(e) => setData(e.target.value)}
        placeholder="Enter sensitive data..."
      />
      <button type="submit" disabled={isProtecting}>
        {isProtecting ? 'Protecting...' : 'Protect Data'}
      </button>
      {protectedId && (
        <div>✅ Protected! ID: {protectedId}</div>
      )}
    </form>
  );
}

Complete Component Example

// SecureNotes.jsx
import React, { useState, useEffect } from 'react';
import { useSoulfra } from '@soulfra/react';

function SecureNotes() {
  const { protect, retrieve, search } = useSoulfra();
  const [notes, setNotes] = useState([]);
  const [newNote, setNewNote] = useState('');
  
  // Load protected notes on mount
  useEffect(() => {
    async function loadNotes() {
      const results = await search('note');
      setNotes(results);
    }
    loadNotes();
  }, []);
  
  const saveNote = async () => {
    const protected = await protect({
      type: 'note',
      content: newNote,
      created: new Date()
    });
    
    setNotes([...notes, protected]);
    setNewNote('');
  };
  
  const viewNote = async (noteId) => {
    const data = await retrieve(noteId);
    alert(data.content);
  };
  
  return (
    <div>
      <h2>Secure Notes</h2>
      
      <div>
        <textarea
          value={newNote}
          onChange={(e) => setNewNote(e.target.value)}
          placeholder="Write a secure note..."
        />
        <button onClick={saveNote}>Save Securely</button>
      </div>
      
      <div>
        <h3>Your Protected Notes</h3>
        {notes.map(note => (
          <div key={note.id}>
            Note from {note.created}
            <button onClick={() => viewNote(note.id)}>
              View
            </button>
          </div>
        ))}
      </div>
    </div>
  );
}
Best Practice: Always clear sensitive data from state after protecting it. Soulfra ensures the data is safe, but keeping it in React state defeats the purpose.
