**Single-command domain deployment system** that creates isolated, production-ready websites automatically.

## Quick Start

### 1. Set Up Credentials

```bash
# Required: GitHub personal access token
export GITHUB_TOKEN="ghp_your_token_here"

# Optional: CloudFlare for automatic DNS
export CLOUDFLARE_TOKEN="your_cloudflare_token"
export CLOUDFLARE_ZONE_ID="your_zone_id"
```

**Get GitHub Token:**
1. Go to https://github.com/settings/tokens
2. Generate new token (classic)
3. Select scopes: `repo`, `admin:org`
4. Copy token and export as shown above

**Get CloudFlare Credentials (optional):**
1. Log into CloudFlare dashboard
2. Go to "My Profile" â†’ "API Tokens"
3. Create token with "Zone:DNS:Edit" permissions
4. Zone ID found in domain overview

### 2. Create a Domain

```bash
# Create a search engine
node create-domain.js deathtodata search-engine

# Create a calendar system
node create-domain.js calriven calendar-system

# Create content moderation platform
node create-domain.js cringeproof content-moderation

# Create generic site
node create-domain.js myapp default
```

### 3. Deploy

The script handles everything automatically:
- Creates repo at `github.com/Soulfra/<domain>.github.io`
- Generates professional HTML/CSS using Ollama
- Pushes code to GitHub
- Enables GitHub Pages
- Configures DNS (if credentials provided)
- Adds to domain portfolio

### 4. Access Your Site

After 1-3 minutes (GitHub Pages build time):
- **GitHub:** `https://github.com/Soulfra/<domain>.github.io`
- **Live Site:** `https://<domain>.com`
- **Portfolio:** `https://soulfra.com` (lists all your domains)

```javascript
const DOMAIN_TEMPLATES = {
  'my-custom-type': {
    description: 'My custom application',

  }

```

### Programmatic Usage

```javascript
const DomainAutomation = require('./create-domain.js');

const automation = new DomainAutomation('myapp', 'search-engine');
await automation.run();
```

### CI/CD Integration

```yaml
# .github/workflows/deploy-domain.yml
name: Deploy New Domain
on:
  workflow_dispatch:
    inputs:
      domain_name:
        description: 'Domain name (e.g., deathtodata)'
        required: true
      domain_type:
        description: 'Domain type'
        required: true
        type: choice
        options:
          - search-engine
          - calendar-system
          - content-moderation
          - default

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Deploy Domain
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          CLOUDFLARE_TOKEN: ${{ secrets.CLOUDFLARE_TOKEN }}
          CLOUDFLARE_ZONE_ID: ${{ secrets.CLOUDFLARE_ZONE_ID }}
        run: |
          node create-domain.js ${{ github.event.inputs.domain_name }} ${{ github.event.inputs.domain_type }}
```

## Troubleshooting

### "GITHUB_TOKEN not set"
```bash
export GITHUB_TOKEN="ghp_your_token_here"
# Or add to ~/.bashrc or ~/.zshrc
```

### "Repository already exists"
The script will continue and update the existing repository. Delete the repo first if you want a fresh start:
```bash
gh repo delete Soulfra/<domain>.github.io --confirm
```

### "Ollama error"
Make sure Ollama is running:
```bash
ollama serve
# In another terminal:
ollama pull llama3.2:latest
```

### DNS Not Working
DNS propagation takes 5-60 minutes. Check status:
```bash
dig <domain>.com
# Should show CNAME to <username>.github.io
```

## Security

- GitHub token stored in environment variable (not in code)
- CloudFlare token stored in environment variable
- All repos can be private if desired (change `private: true` in script)
- HTTPS enforced by GitHub Pages
- No secrets committed to repositories
